# `verify/` — verification instruments

Re-runnable checks for the `verify:` rigor-debt markers in the physics docs. **The
instruments are source** (version-controlled, re-runnable); **their verdicts are cache**
(disposable, regenerated by re-running) — see `CONVENTIONS.md` §3. Nothing here is pasted
back into the prose as ground truth; findings annotate `docs/rigor-debt.md`, and discharged
claims get an inline `verified:` *attestation* (signature only, never the result).

## Running

Each script is a standalone [PEP 723](https://peps.python.org/pep-0723/) file with inline
dependencies, so `uv` provisions SymPy automatically:

```sh
uv run verify/resogram_edot.py        # one claim
for f in sol edot drive eincr esol; do uv run verify/resogram_$f.py; done   # all five
```

Every script prints a `VERDICT:` line (`✓` / `✗` / inconclusive) and, for confirmed
claims, a `CLAIM_HASH8`.

## `verified:` attestation hashes

A discharged claim carries an inline marker

```
<!-- verified:<tier> claim=<srepr-hash8> by=<instrument>@<filehash8> -->
```

a **composite key** (Nix-derivation style): either the claim *or* the instrument changing
invalidates it.

- `claim=` — first 8 hex of `sha256(srepr(<canonical sympy claim>))`. The script prints this
  as `CLAIM_HASH8`. Using SymPy's `srepr` of the *sympified* claim means cosmetic LaTeX edits
  don't false-trigger; a genuine change to the statement does.
- `@<filehash8>` — first 8 hex of `sha256(<instrument file bytes>)`:
  `sha256sum verify/resogram_<h>.py | cut -c1-8`.

Recompute to check for drift:

```sh
uv run verify/resogram_drive.py | grep CLAIM_HASH8        # claim hash
sha256sum verify/resogram_drive.py | cut -c1-8            # instrument hash
```

> **Scope note (N=2, deferred).** An *automated* source-walking staleness checker — one that
> parses the markers, re-derives both hashes, and flags drift across the whole tree — is
> deliberately **not** built yet (`TODO.md` id `04bb`, gated on the acoustics pilot as the
> second consumer). The grammar above is fixed now so that checker is mechanical to add later.

harness (Lean4 + Mathlib via `lake`) is scoped as a separate, heavier session (`TODO.md` id
